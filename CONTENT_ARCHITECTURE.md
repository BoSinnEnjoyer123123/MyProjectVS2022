# Контентная архитектура новой версии сайта

## 1. Рабочее решение

Основной фокус сайта: пильные инструменты, станки и ленточные пилы для мясопереработки.

Главное направление:

- ленточные пилы для мяса;
- пильные полотна к станкам;
- станки для разделки мяса, птицы, туш и полутуш;
- решения для замороженной продукции;
- оборудование и расходные материалы для снятия кожи с туш.

Кормовые добавки и старый агро-ассортимент остаются только как дополнительное направление. Они не должны появляться в первом экране и не должны конкурировать с главным направлением.

## 2. Цель страницы

За первые 5-7 секунд посетитель должен понять:

- компания занимается пильными инструментами и оборудованием для мясопереработки;
- можно подобрать ленточную пилу или полотно под конкретный станок;
- решения подходят для мяса, кости, курицы, птицы, туш, полутуш и заморозки;
- есть отдельное направление по снятию кожи с туш;
- можно быстро отправить параметры и получить подбор.

Главное действие пользователя: отправить параметры станка/полотна или позвонить.

## 3. Целевая аудитория

Основные пользователи:

- мясоперерабатывающие предприятия;
- цеха разделки мяса;
- птицеперерабатывающие производства;
- магазины с мясным отделом;
- фермерские хозяйства с собственной разделкой;
- сервисные специалисты;
- закупщики расходных материалов.

Что им важно:

- подобрать полотно под станок;
- узнать, подходит ли полотно для мяса, кости, птицы или заморозки;
- быстро отправить параметры;
- получить понятный ответ по наличию, срокам и поставке;
- видеть современный, технически надежный сайт.

## 20. Следующее развитие структуры

После базовой переработки сайт должен лучше конвертировать посетителя в конкретную заявку. Главный новый блок - "Заявка на подбор полотна".

### 20.1. Блок "Заявка на подбор полотна"

Разместить после общего описания подбора полотна на главной и на странице `pily.html`.

Заголовок:

```text
Что указать в заявке на подбор полотна
```

Короткий текст:

```text
Можно отправить параметры письмом или приложить фото станка, шильдика и старого полотна. Чем точнее данные, тем быстрее подбор.
```

Таблица параметров:

| Параметр | Что указать |
|---|---|
| Модель станка | Производитель, модель или фото шильдика |
| Длина полотна | Фактическая длина старого полотна или данные из паспорта |
| Ширина и толщина | Если известны, указать в миллиметрах |
| Шаг зуба | Если неизвестен, можно отправить фото полотна |
| Продукт | Мясо, кость, птица, туши, полутуши, заморозка |
| Режим работы | Объем, сменность, частота замены |
| Количество | Сколько полотен нужно сейчас или в месяц |
| Город поставки | Для расчета доставки и сроков |

CTA:

```text
Отправить параметры полотна
Отправить фото станка
Позвонить для подбора
```

### 20.2. Роль блока

Блок должен:

- уменьшить количество неполных обращений;
- объяснить, что не обязательно знать все параметры;
- показать, что можно начать подбор по фото;
- подвести к email/телефону;
- усилить ощущение экспертного подбора, а не просто продажи "пил".

### 20.3. Следующие блоки после заявки

После блока заявки можно развивать:

- микро-каталог полотен по задачам;
- блок доверия и условий поставки;
- блок совместимости со станками;
- более точное направление снятия кожи;
- аналитику кликов по телефону и email.
