# План радикальной переработки сайта-визитки для GitHub Pages

## 1. Цель переработки

Переделать текущий сайт из чернового каталога в понятную B2B-визитку компании, которая быстро отвечает на вопросы посетителя:

- что компания поставляет;
- кому это подходит;
- от какого объема работает;
- как быстро связаться;
- какие товары доступны;
- почему можно доверять поставщику.

Сайт должен быть статическим и размещаться на бесплатном GitHub Pages без серверной части.

## 2. Текущее состояние

Сейчас сайт находится в папке `Test123` и состоит из:

- `index.html` - главная страница с описанием компании и ассортиментом;
- `pila.html` - отдельная страница про пилы;
- `css/styles.css` - общий файл стилей;
- `js/script.js` - бургер-меню, карусели, тема, кнопка наверх;
- папки `images` с большим количеством PNG-изображений;
- архива `Test123.zip`, который не должен попадать в публикацию.

Основные проблемы:

- первый экран не продает: начинается с блока "О компании", нет сильного оффера;
- контакты спрятаны в выпадающий блок, нет явных быстрых действий;
- мобильная шапка слишком высокая и занимает значительную часть экрана;
- подменю каталога завязано на hover, что плохо работает на телефонах;
- автокарусели усложняют страницу и не дают заметной пользы;
- карточки товаров малоинформативны: нет условий поставки, фасовки, документов, CTA;
- раздел "Пилы" конфликтует с основной тематикой кормовых ингредиентов;
- изображения тяжелые, суммарно около 55 МБ;
- внутри папки сайта есть `Test123.zip` размером около 55 МБ;
- CSS содержит много повторяющихся правил и финальных override-блоков;
- структура проекта не подготовлена аккуратно под GitHub Pages.

## 3. Целевая концепция

Новый сайт должен восприниматься как деловая страница поставщика кормовых ингредиентов и удобрений.

Главная идея:

> Поставка кормовых ингредиентов и удобрений партиями от 20 тонн с подбором под задачу клиента.

Тон сайта:

- деловой;
- простой;
- без маркетингового шума;
- с акцентом на поставку, объемы, ассортимент и контакт;
- без лишней декоративности.

Визуальное направление:

- чистая B2B-верстка;
- светлая основная тема;
- зеленый как акцент, но не полностью зеленая страница;
- плотные, аккуратные блоки;
- минимум "игрушечных" эффектов;
- больше пользы в тексте и карточках.

## 4. Новая структура сайта

Рекомендуемая структура для первой версии:

```text
/
├── index.html
├── 404.html
├── assets/
│   ├── css/
│   │   └── styles.css
│   ├── js/
│   │   └── main.js
│   ├── img/
│   │   ├── logo.webp
│   │   ├── hero.webp
│   │   ├── products/
│   │   └── icons/
│   └── favicon/
├── README.md
└── REDESIGN_PLAN.md
```

`ConsoleApp1` и прочие не относящиеся к сайту файлы не должны участвовать в публикации GitHub Pages. Если репозиторий остается общим, для Pages лучше настроить публикацию из отдельной папки, например `/docs`, или перенести сайт в корень репозитория.

## 5. Рекомендуемая карта страницы

### 5.1. Header

Содержимое:

- логотип;
- навигация: "Ассортимент", "Условия", "Преимущества", "Контакты";
- видимый телефон;
- кнопка "Связаться".

Поведение:

- на desktop шапка компактная, высотой примерно 72-88 px;
- на mobile логотип, телефон/иконка звонка и бургер в одну строку;
- мобильное меню открывается по клику;
- все пункты меню доступны без hover;
- шапка может быть sticky, но не должна занимать много места.

Убрать из первого релиза:

- переключатель темной темы, если он не несет бизнес-пользы;
- сложное выпадающее меню;
- большой логотип высотой 120 px.

### 5.2. Hero-блок

Первый экран должен сразу объяснять предложение.

Пример содержания:

```text
Поставка кормовых ингредиентов и удобрений от 20 тонн
Кормовые дрожжи, барда, жмых, люпин, минеральные удобрения и добавки для сельхозпредприятий и производств.
Подберем позицию под задачу, согласуем объем и условия отгрузки.
```

CTA:

- "Позвонить";
- "Написать в WhatsApp";
- "Запросить условия поставки".

Факты рядом с CTA:

- минимальная партия от 20 тонн;
- регулярные поставки;
- работа с проверенными поставщиками;
- подбор под потребность заказчика.

### 5.3. Блок "Ассортимент"

Оформить карточками 2-3 в ряд на desktop и по 1 на mobile.

Карточки:

- кормовые дрожжи;
- барда послеспиртовая;
- жмых рапсовый;
- жмых подсолнечный;
- люпин;
- минеральные удобрения;
- кормовые добавки.

Для каждой карточки:

- название;
- 1 качественное изображение;
- краткое назначение;
- форма поставки, если известна;
- минимальная партия;
- кнопка "Запросить условия".

Карусели в карточках убрать или заменить на простую галерею, открывающуюся по клику. Для сайта-визитки автокарусель не обязательна.

### 5.4. Блок "Условия поставки"

Задача блока - снять основные вопросы клиента до звонка.

Содержимое:

- минимальный объем: от 20 тонн;
- возможные форматы отгрузки: навалом, биг-беги, мешки, если применимо;
- география поставок;
- порядок работы:
  1. клиент отправляет запрос;
  2. компания уточняет позицию, объем, регион;
  3. согласуются цена и сроки;
  4. оформляется поставка.

Если точных данных пока нет, оставить нейтральные формулировки без выдуманных гарантий.

### 5.5. Блок "Преимущества"

Не делать абстрактные фразы вроде "лучшее качество". Нужны проверяемые преимущества:

- работаем с проверенными поставщиками и производителями;
- поставки на постоянной основе;
- подбор продукции под потребности заказчика;
- минимальная партия от 20 тонн;
- понятная коммуникация по телефону и email.

### 5.6. Блок "Контакты"

Контакты должны быть видны и кликабельны.

Содержимое:

- телефон 1: `+7 908 585-44-45`;
- телефон 2: `+7 908 080-20-60`;
- email: `agrokom74@gmail.com`;
- email: `agrokom174@gmail.com`;
- кнопка WhatsApp, если номер используется в WhatsApp;
- кнопка Telegram, если есть аккаунт;
- часы ответа, если известны;
- регион или город, если его можно указывать.

Технические ссылки:

```html

<a href="mailto:agrokom74@gmail.com">agrokom74@gmail.com</a>
```

### 5.7. Footer

Содержимое:

- название компании;
- год;
- основные контакты;
- повтор навигации;
- дисклеймер об иллюстративных изображениях, если изображения сгенерированы ИИ;

## 6. Что делать со страницей "Пилы"

Сейчас `pila.html` выглядит как чужеродная страница: основной сайт про кормовые ингредиенты и удобрения, а "Пилы" относятся к другому направлению.

Варианты:

### Вариант A - убрать из первой версии

Рекомендуется, если пилы не являются ключевым направлением бизнеса.

Плюсы:

- сайт становится сфокусированным;
- проще объяснить предложение;
- меньше лишних страниц;
- лучше для SEO и доверия.

### Вариант B - оставить как отдельное направление

Если пилы действительно продаются, нужно оформить их как отдельный раздел:

- добавить в hero или навигацию формулировку "Кормовые ингредиенты, удобрения и пильные материалы";
- сделать отдельный блок "Пильные материалы";
- добавить характеристики: типы пил, материалы, размеры, поставка под заказ;
- добавить CTA;
- не смешивать карточку пил с кормовыми товарами.

Для радикальной переработки лучше выбрать один фокус. Если фокус - агро и корма, `pila.html` временно убрать из публичной навигации.

### Принятое решение после первой публикации

Направление "Пилы" возвращается на сайт как отдельное ответвление, а не как часть ассортимента кормовых ингредиентов.

Реализация:

- на главной странице добавить отдельный блок "Дополнительное направление: пильные материалы";
- вести из этого блока и из навигации на отдельную страницу `pily.html`;
- не добавлять пилы в карточки раздела "Ассортимент";
- на странице `pily.html` использовать отдельный hero, параметры подбора, форматы пильных материалов и контакты;
- использовать те же телефоны и email, но с темой письма по пильным материалам;
- визуально сохранить общий стиль сайта, чтобы направление выглядело частью компании, но не смешивалось с агро-каталогом.

Обновленная структура:

```text
Главная
├── Кормовые ингредиенты и удобрения
├── Дополнительное направление: пильные материалы
└── Контакты

pily.html
├── Hero по пильным материалам
├── Параметры подбора
├── Форматы: пильные ленты, диски, подбор под задачу
├── Процесс запроса
└── Контакты
```

Поставка кормовых ингредиентов и удобрений от 20 тонн

Текущий текст можно сократить и перенести ниже.

ООО "Компаньон" поставляет кормовые ингредиенты, минеральные удобрения и кормовые добавки для сельхозпредприятий, производств и оптовых заказчиков. Работаем с проверенными поставщиками и подбираем продукцию под объем, регион и задачу клиента.

Кормовые дрожжи

Использовать конкретные действия:

- "Позвонить";
- "Написать в WhatsApp";
- "Запросить цену";
- "Уточнить наличие";
- "Обсудить поставку".

Не использовать расплывчатое:

- "Подробнее";
- "Узнать больше";
- "Отправить".

## 8. Дизайн-система

### 8.1. Цвета

Базовая палитра:

```css
:root {
  --color-bg: #f7f8f5;
  --color-surface: #ffffff;
  --color-text: #1f2a24;
  --color-muted: #65736b;
  --color-brand: #2f7d46;
  --color-brand-dark: #205c33;
  --color-accent: #d6a73a;
  --color-border: #dfe5dd;
}
```

Важно: не делать весь сайт зеленым. Зеленый использовать для акцентов, кнопок и заголовков.

### 8.2. Типографика

Рекомендации:

- использовать системный стек или один webfont;
- не подключать лишние шрифты;
- базовый текст 16-18 px;
- заголовки без чрезмерного размера;
- нормальная межстрочная высота 1.45-1.6.

Пример:

```css
body {
  font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
}
```

### 8.3. Сетка

Desktop:

- контейнер 1120-1200 px;
- карточки товаров 3 колонки;
- большие блоки с вертикальным отступом 64-88 px.

Tablet:

- 2 колонки карточек;
- шапка остается компактной.

Mobile:

- 1 колонка;
- CTA-кнопки во всю ширину;
- шапка не выше 72-88 px в закрытом состоянии.

### 8.4. Компоненты

Нужны компоненты:

- header;
- mobile menu;
- hero;
- кнопка;
- карточка товара;
- блок фактов;
- блок этапов поставки;
- контактная панель;
- footer.

Не нужны в первой версии:

- темная тема;
- автокарусели;
- сложные выпадающие меню;
- декоративные анимации;
- большие фоновые градиенты.

## 9. Технические требования

### 9.1. GitHub Pages

Сайт должен работать на бесплатном GitHub Pages.

Требования:

- все пути относительные;
- главная страница называется `index.html`;
- не использовать backend;
- не использовать PHP, Node-сервер, базы данных;
- формы либо не использовать, либо подключать внешний сервис;
- не хранить лишние архивы и исходники в папке публикации.

Варианты публикации:

1. Публиковать из корня репозитория.
2. Публиковать из папки `/docs`.

Рекомендуемый вариант - `/docs`, если в репозитории остаются посторонние файлы вроде `ConsoleApp1`.

### 9.2. HTML

Требования:

- семантическая структура: `header`, `main`, `section`, `article`, `footer`;
- один `h1` на страницу;
- корректный `lang="ru"`;
- нормальные `title` и `meta description`;
- alt-тексты для изображений;
- ссылки `tel:` и `mailto:`;
- без мусорных пустых блоков;
- без inline-стилей.

### 9.3. CSS

Требования:

- один чистый файл стилей;
- убрать дублирующиеся правила;
- убрать конфликтующие override-блоки;
- использовать CSS-переменные;
- mobile-first или аккуратные breakpoint-правила;
- не использовать отрицательные letter-spacing;
- не допускать горизонтального скролла;
- фиксировать размеры ключевых элементов: логотип, кнопки, карточки, изображения.

### 9.4. JavaScript

Нужен минимальный JS:

- открытие/закрытие мобильного меню;
- закрытие меню по клику на пункт;
- кнопка "Наверх", если нужна;
- простая галерея, если оставляем несколько изображений.

Убрать или не делать:

- автоперелистывание каруселей;
- переключатель темной темы;
- сложную логику, не влияющую на заявку.

### 9.5. Производительность

Цели:

- вес главной страницы с ассетами первого экрана до 1.5-2 МБ;
- общий вес изображений в публикации желательно до 5-8 МБ;
- изображения в WebP или AVIF;
- lazy loading для изображений ниже первого экрана;
- logo не больше 100-200 КБ;
- hero-изображение не больше 300-500 КБ;
- удалить `Test123.zip` из папки сайта.

Текущие PNG по 2-3 МБ нужно пересжать.

### 9.6. SEO и предпросмотр ссылок

Добавить:

```html
<title>Поставка кормовых ингредиентов и удобрений от 20 тонн | Агрокомпаньон</title>
<meta name="description" content="Поставка кормовых дрожжей, барды, жмыха, люпина, минеральных удобрений и кормовых добавок партиями от 20 тонн.">
<meta property="og:title" content="Поставка кормовых ингредиентов и удобрений от 20 тонн">
<meta property="og:description" content="Кормовые ингредиенты, удобрения и добавки для сельхозпредприятий и оптовых заказчиков.">
<meta property="og:type" content="website">
<meta property="og:image" content="assets/img/og-image.webp">
```

Добавить favicon:

- `favicon.ico`;
- `apple-touch-icon.png`;
- `site.webmanifest`, если требуется.

## 10. Доступность

Требования:

- все интерактивные элементы доступны с клавиатуры;
- видимый focus state;
- кнопки имеют понятный текст;
- иконки не заменяют текст там, где действие может быть неочевидно;
- контраст текста не ниже WCAG AA;
- мобильное меню имеет `aria-expanded`;
- изображения имеют полезные `alt`, если несут смысл.

## 11. План работ

### Этап 1. Подготовка проекта

Задачи:

- выбрать схему публикации: корень или `/docs`;
- убрать из публикуемой папки `Test123.zip`;
- отделить сайт от `ConsoleApp1`;
- создать новую структуру `assets/css`, `assets/js`, `assets/img`;
- перенести только нужные файлы.

Результат:

- чистая структура сайта;
- нет лишних архивов;
- понятно, что именно публикуется на GitHub Pages.

### Этап 2. Контент и информационная архитектура

Задачи:

- определить основной фокус сайта: агро/корма или смешанный каталог;
- решить судьбу страницы `pila.html`;
- переписать hero;
- переписать описание компании;
- подготовить тексты карточек товаров;
- подготовить блок условий поставки;
- подготовить контакты и CTA.

Результат:

- готовая структура текста;
- понятная логика страницы;
- нет случайных разделов.

### Этап 3. Новый HTML

Задачи:

- создать новый `index.html`;
- сделать семантические секции;
- добавить кликабельные контакты;
- добавить SEO-мета;
- добавить Open Graph;
- добавить footer;
- добавить `404.html`.

Результат:

- чистая HTML-страница без старой накопленной разметки;
- сайт открывается локально и через GitHub Pages.

### Этап 4. Новый CSS

Задачи:

- написать стили с нуля;
- настроить палитру;
- сделать компактную шапку;
- сверстать hero;
- сверстать карточки ассортимента;
- сверстать блок условий и контактов;
- настроить mobile/tablet/desktop;
- проверить отсутствие горизонтального скролла.

Результат:

- единый, поддерживаемый CSS;
- нормальная адаптивность;
- аккуратный внешний вид.

### Этап 5. Изображения

Задачи:

- выбрать по 1 основному изображению на товар;
- удалить дубли и неудачные изображения;
- конвертировать PNG в WebP;
- подготовить размеры:
  - logo: до 240 px по ширине;
  - product images: примерно 800x600 или 900x675;
  - og-image: 1200x630;
  - hero: около 1400x900, если нужен;
- прописать `width`, `height`, `loading="lazy"`.

Результат:

- сайт загружается значительно быстрее;
- изображения выглядят ровно в карточках;
- репозиторий не раздувается.

### Этап 6. JavaScript

Задачи:

- оставить только нужную логику;
- реализовать мобильное меню;
- закрывать меню по клику вне области или по пункту;
- добавить кнопку "Наверх" при необходимости;
- если нужна галерея, сделать простую без автолистания.

Результат:

- JS короткий и понятный;
- сайт не зависит от сложной логики.

### Этап 7. Проверка

Проверить:

- desktop 1280x720;
- desktop 1440x900;
- mobile 390x844;
- mobile 360x740;
- все ссылки меню;
- `tel:` ссылки;
- `mailto:` ссылки;
- отсутствие 404 на ассеты;
- отсутствие горизонтального скролла;
- скорость загрузки;
- отображение на GitHub Pages.

Критерии:

- первый экран понятен без прокрутки;
- контакты доступны за 1 действие;
- карточки товаров читаются на телефоне;
- сайт не ломается без серверной части;
- общий вес публикации адекватен для визитки.

## 12. Критерии приемки

Сайт считается готовым, если:

- размещается на GitHub Pages;
- главная страница открывается по адресу Pages без дополнительных настроек сервера;
- первый экран содержит оффер, преимущества и CTA;
- меню работает на desktop и mobile;
- все контакты кликабельны;
- изображения оптимизированы;
- нет архива `Test123.zip` в публикуемой папке;
- нет горизонтального скролла на мобильных;
- нет битых изображений;
- Lighthouse Performance желательно выше 80 на mobile;
- Lighthouse Accessibility желательно выше 90;
- HTML и CSS не содержат старых дублирующих блоков;
- сайт можно поддерживать вручную без сборщика.

## 13. Приоритеты реализации

- удаление лишнего zip-архива;

## 14. Предлагаемый результат первой версии

Первая радикально переработанная версия должна быть одним быстрым статическим лендингом:

```text
Header
Hero с оффером и CTA
Факты о поставке
Ассортимент
Условия работы
Преимущества
Контакты
Footer
```

Это даст максимум пользы при минимальной сложности и хорошо подходит для бесплатного GitHub Pages.
